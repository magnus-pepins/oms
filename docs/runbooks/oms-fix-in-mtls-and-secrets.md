# FIX-in mTLS and credential rotation

Operator runbook for **oms-fix-ingress** acceptor TLS and staged logon secrets. Complements [oms-fix-ingress.md](./oms-fix-ingress.md).

## mTLS (production)

1. Terminate client TLS at the edge **or** enable QuickFIX/J SSL settings in the acceptor `SessionSettings` file generated by `FixInSessionSettingsLoader` (extend with `SocketUseSSL=Y`, cert paths per session).
2. Store private keys in the platform secret manager (not git). Mount read-only into the fix-ingress container host path referenced by env.
3. Client certificate DN / SAN must match the row in `oms_fix_in_counterparty` (future: `client_cert_subject` column). Until wired, enforce IP allow-list + username/password on logon.
4. Rotate server cert: rolling restart of **only** `oms-fix-ingress` PM2 role after dual-trust window; never restart the Aeron cluster for cert rotation.

## Staged credential rotation

1. Ops Console → Trading → Sessions → select FIX-in session → **Credential rotation** (or `POST /internal/v1/fix-in/sessions/{id}/credential-rotation` with `newPasswordHash`).
2. Hash is stored in `oms_fix_in_session.password_hash` (plaintext comparison today when `OMS_FIX_IN_REQUIRE_LOGON_CREDENTIALS=true` — replace with bcrypt before production).
3. Row is appended to `oms_fix_session_admin_actions` (`CREDENTIAL_ROTATION`).
4. **Force logout** the session, confirm counterparty applies the new password on next logon, smoke one `D` message.

## Sequence reset (audited)

1. Disable session or **force logout** until `loggedOn=false`.
2. Agree `nextSenderSeq` / `nextTargetSeq` with counterparty (reference ticket in `counterpartyReference`).
3. `POST .../sequence-reset` — writes `oms_fix_session_admin_actions` and mutates QuickFIX store.
4. Re-enable and logon; verify seq nums in Ops Console table match counterparty.

## Rate limits

| Env | Default | Purpose |
|-----|---------|---------|
| `OMS_FIX_IN_MAX_MESSAGE_AGE_MS` | 120000 | Reject stale `SendingTime` |
| `OMS_FIX_IN_MAX_CL_ORD_ID_LEN` | 64 | Reject oversized ClOrdID |
| `OMS_FIX_IN_MAX_APP_MSG_PER_SEC` | 200 | Per-wire-session token bucket |

Bench automation: `./gradlew fixInLoopbackConformanceProbe` scenario **11** uses stale `SendingTime` (~125s); QuickFIX may reject at session layer before app `message_too_old`. Burst testing requires lowering `OMS_FIX_IN_MAX_APP_MSG_PER_SEC` temporarily.

Metrics: `oms_fix_in_rate_limit_rejects_total`, `oms_fix_in_audit_write_failures_total`.

## Raw message retention

- Metadata: `oms_fix_message_audit` (summary + optional `raw_store_ref` → `oms_fix_messages` when JDBC store).
- Retention hint: `OMS_FIX_IN_MESSAGE_AUDIT_RETENTION_DAYS` (default 90). Operator cleanup:

```sql
DELETE FROM oms_fix_message_audit WHERE created_at < NOW() - INTERVAL '90 days';
```

## Ops Console RBAC

Map Supabase / beard-internal role `trading.fix_admin` → ops-console `fix_admin` for sequence reset, logout, credential rotation. `operator` retains read-only on `/api/oms-trading/fix/*`.
